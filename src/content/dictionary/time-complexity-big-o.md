---
term: "Time Complexity (Big O)"
category: "Algorithms"
level: "Foundational"
---

A mathematical notation describing how an algorithm's execution time grows as input size increases. Focuses on the dominant term.

### Example

O(1) = instant. O(log n) = binary search. O(n) = scan a list. O(n²) = nested loops. O(2ⁿ) = brute-force subsets. Tells you if it'll scale.
