---
term: "Test-Driven Development (TDD)"
category: "Testing & QA"
level: "Intermediate"
---

Write tests BEFORE writing code. Red-Green-Refactor cycle: write a failing test (red), write minimal code to pass (green), refactor.

### Example

Step 1: write test 'add(2,3) should return 5' — it fails (function doesn't exist). Step 2: write add(a,b) { return a+b } — test passes. Step 3: refactor if needed.
